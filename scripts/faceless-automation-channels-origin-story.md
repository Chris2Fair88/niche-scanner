## Production notes, tied to the free-tier stack we already set up

- **Screen recording:** CapCut's built-in screen recorder (or OBS if you want more control) at 1080x1920. Terminal/GitHub windows are natively horizontal — either record vertical directly with a narrow terminal window, or record normally and reframe/zoom to vertical crop in CapCut's Shorts editor, focused on the relevant text each time.
- **Voiceover:** ElevenLabs, free tier. Go for a clear, confident, slightly fast-paced preset — this is tech/build-in-public content, so a flat "corporate narrator" voice undersells the story beats. Worth a couple of preset auditions before committing.
- **Music:** Use CapCut's built-in royalty-free library (search "tech" or "build" mood), not an external licensed track — this keeps you in the higher Shorts revenue-share tier we discussed (original/library audio vs. licensed music).
- **Captions:** CapCut auto-captions, styled bold and high-contrast — most Shorts get watched muted, so the caption is doing real work, not just decoration. Match the teal/white from the channel branding for consistency.
- **Disclosure:** Since this uses AI voice, check the "altered or synthetic content" box in YouTube Studio when uploading — one click, and it's the thing that keeps you clean on the policy front we discussed early on.
